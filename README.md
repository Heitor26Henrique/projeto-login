### Git

O Git é uma ferramenta usada para controlar versões de um projeto, permitindo acompanhar todas as alterações feitas nos arquivos ao longo do tempo. Com ele, você consegue saber o que foi modificado, quando foi modificado e até voltar para versões anteriores caso aconteça algum erro. Isso traz mais segurança, organização e facilita muito o desenvolvimento, principalmente em equipe.

### git status

O comando git status é utilizado para verificar a situação atual do projeto. Ele mostra quais arquivos foram alterados, quais ainda não estão sendo controlados pelo Git e quais já estão preparados para serem salvos. É como olhar um painel de controle antes de decidir o que fazer com as mudanças.

### git add

O git add serve para selecionar quais alterações você quer registrar. Quando você modifica um arquivo, o Git não salva automaticamente; é necessário indicar quais mudanças devem fazer parte da próxima versão. Esse comando funciona como separar os documentos que você deseja guardar antes de arquivar.

### git commit -m

O git commit -m é o momento em que as alterações são realmente salvas no histórico do projeto. Ao fazer um commit, você cria um registro daquele ponto do trabalho, acompanhado de uma mensagem explicando o que foi feito. Essa descrição ajuda a entender a evolução do sistema e mantém tudo bem documentado.

### git push

O git push envia as alterações salvas para um repositório remoto, como o GitHub. Até esse momento, tudo estava apenas no seu computador; com o push, o projeto é atualizado na internet, permitindo que outras pessoas tenham acesso às mudanças e continuem o trabalho de forma colaborativa.
